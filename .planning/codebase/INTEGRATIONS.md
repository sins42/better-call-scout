# External Integrations

**Analysis Date:** 2026-04-03

## APIs & External Services

**GitHub:**
- GitHub REST API — Searches repos by topic, language, and star count; fetches star history, commit activity, contributor stats, and issue velocity
  - SDK/Client: `requests` (direct HTTP calls)
  - Auth: `GITHUB_TOKEN` (Personal Access Token, set in `.env`)
  - Consuming agent: `src/agents/collection/github_agent.py`
  - Key endpoints (inferred): `GET /search/repositories`, `GET /repos/{owner}/{repo}/stargazers`, `GET /repos/{owner}/{repo}/stats/commit_activity`, `GET /repos/{owner}/{repo}/contributors`, `GET /repos/{owner}/{repo}/issues`

**Hacker News:**
- HN Firebase REST API — Fetches top/best stories for startup and technology signal
  - SDK/Client: `requests` (direct HTTP calls to Firebase-backed HN API at `https://hacker-news.firebaseio.com/v0/`)
  - Auth: None required (public API)
  - Consuming agents: `src/agents/collection/hn_tavily_agent.py`, `src/rag/ingestion.py`
  - Also used in RAG ingestion: HN stories are fetched, chunked, embedded, and stored in ChromaDB

**Tavily:**
- Tavily Search API — Web search for founder bios, funding news, and job postings enrichment
  - SDK/Client: `tavily-python >= 0.3.0`
  - Auth: `TAVILY_API_KEY` (set in `.env`)
  - Consuming agent: `src/agents/collection/hn_tavily_agent.py`

**Google Vertex AI / Gemini:**
- Vertex AI — Provides Gemini model inference for all analyst and synthesis agents
  - SDK/Client: `google-cloud-aiplatform >= 1.60.0`
  - Auth: `GOOGLE_CLOUD_PROJECT` (GCP project ID set in `.env`); Application Default Credentials (ADC) assumed for service account auth in Cloud Run
  - Consuming agents: `src/agents/analysis/vc_analyst.py`, `src/agents/analysis/developer_analyst.py`, `src/agents/analysis/journalist_analyst.py`, `src/agents/synthesis_agent.py`, `src/orchestrator.py`
  - Usage: Analyst agents emit structured hypothesis JSON; synthesis agent merges them into a unified scout report

**Google ADK:**
- Google Agent Development Kit — Orchestrates the multi-agent pipeline; manages parallel execution of collection and analysis layers
  - SDK/Client: `google-adk >= 0.1.0`
  - Auth: Inherits Vertex AI / GCP credentials
  - Entry point: `src/orchestrator.py`
  - Pipeline flow: Collection layer -> Critic -> Analysis layer (parallel) -> Synthesis

**RSS Feeds:**
- Generic RSS — Fetched during RAG ingestion alongside HN stories to populate the ChromaDB corpus with domain context
  - SDK/Client: `requests` (direct HTTP, no dedicated RSS library detected)
  - Auth: None required
  - Consuming module: `src/rag/ingestion.py`

## Data Storage

**Databases:**
- ChromaDB (embedded vector store)
  - Type: In-process, file-backed vector database
  - Client: `chromadb >= 0.5.0`
  - Connection: No network connection; runs embedded within the Python process
  - No env var for connection string (path is configured in application code)
  - Populated by: `src/rag/ingestion.py` (chunks + MiniLM embeddings of HN stories and RSS feeds)
  - Queried by: `src/rag/retrieval.py`, `src/agents/collection/rag_agent.py`
  - Note: ChromaDB data is not persisted across container restarts unless a volume is mounted; ingestion must run before retrieval queries are meaningful

**File Storage:**
- Local filesystem only (within container)
  - `scout_report.md` — Markdown report artifact generated by `src/agents/synthesis_agent.py`
  - `top_repos.csv` — CSV artifact generated by `src/agents/synthesis_agent.py`
  - Both are surfaced as download buttons in `app/streamlit_app.py`
