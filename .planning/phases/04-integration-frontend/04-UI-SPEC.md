## Layout Contract

Single scrolling page. No routing. No SPA framework. Sections stack vertically in this order:

1. **Header bar** — fixed top, 56px height. Left: site name (Display 28px). Right: "Run" primary button.
2. **Query card** — white card with border, 16px inner padding. Contains: text input (full width), persona checkboxes row below (VC / Developer / Journalist, default all checked).
3. **SSE progress strip** — hidden until run starts (`display: none` → `display: flex` on first SSE event). Shows 4 breadcrumb stages: Collection → Critic → Analysis → Synthesis. Each stage: Label text at Heading 20px, color shifts from text-secondary (#6b7280) to VC accent (#6366f1, used as pipeline accent) as it completes.
4. **Persona pill tabs + result card** — hidden until pipeline complete. Pill tabs row (one per selected persona). Active tab shows persona accent color. Below: hypothesis text (Body 16px), confidence dot-rating (●●●●○ 82%), collapsible evidence/counter-evidence list.
5. **Download bar** — sits above charts panel. Contains: "Download Report" button (scout_report.md), "Download CSV" button (top_repos.csv), "Download Charts" buttons (individual PNG per chart, 4 buttons). Buttons are secondary style (border, no fill).
6. **Charts panel** — 2×2 CSS grid of 4 `<img>` tags, each fetched from `GET /download/chart_{n}.png`. Collapses to 1-column below 640px breakpoint.

**Source:** CONTEXT.md D-12, D-13, D-14, D-15.

## Component Interaction Contract

### Run Button
- **Default state:** Enabled. Label: "Run Scout" (Label 14px, weight 600). Background: #6366f1 (VC indigo used as primary action color). Text: #ffffff. Border-radius: 8px. Padding: 8px 16px.
- **Loading state:** Disabled (pointer-events: none, opacity: 0.6). Label: "Running…". Do not show spinner — SSE strip handles visual progress.
- **Post-run state:** Re-enabled. Label: "Run Scout".
- **On error:** Re-enabled immediately. Label: "Run Scout". Error appears inline below query card.

### Persona Checkboxes
- Standard HTML checkboxes with custom CSS. min-height: 44px per row.
- Label: persona name at Label 14px, weight 600.
- All 3 checked by default.
- Unchecking a persona hides that tab from the results section. Data still fetched.

### SSE Progress Strip (breadcrumb)
- 4 stages displayed left to right: "Collection" → "Critic" → "Analysis" → "Synthesis"
- Inactive stage: text-secondary (#6b7280), no fill.
- Active (in progress) stage: text-primary (#111827), animated pulse (CSS `@keyframes pulse { 0%,100%{opacity:1} 50%{opacity:0.5} }`, 1.2s infinite, applied to stage label only).
- Completed stage: #6366f1 (pipeline accent), no animation. A checkmark glyph (✓) prepended to stage label.
- Strip hides after 2s delay once "Synthesis complete" event received and results section becomes visible.

**Source:** CONTEXT.md D-07. Animation style is Claude's discretion — pulse chosen for clarity over fade-in (fade-in can be mistaken for completion).

### Persona Pill Tabs
- Shape: fully rounded pill (border-radius: 9999px). Padding: 8px 20px. Gap between pills: 8px.
- Inactive: background #f3f4f6, text text-secondary (#6b7280), border 1px solid #e5e7eb.
- Active: background = persona accent color at 15% opacity, text = persona accent color, border 1.5px solid persona accent color.
- Click switches active tab; only one tab active at a time.

### Confidence Dot Rating
- Rendered as Unicode filled/empty circles: ● (U+25CF) and ○ (U+25CB).
- Scale: 5 dots. Filled dots = floor(confidence_score / 20). E.g., score 82 → 4 filled, 1 empty: ●●●●○ 82%.
- Color: persona accent color for filled dots, text-secondary for empty.
- Displayed inline after hypothesis text block.

### Collapsible Evidence List
- Collapsed by default. Toggle label: "Show evidence ▼" / "Hide evidence ▲" (Label 14px, text-secondary).
- Expanded: ordered list of evidence strings, each at Body 16px. Counter-evidence section below with label "Counter-evidence" at Heading 20px, weight 600, text-secondary.
- No animation required — instant show/hide via `display` toggle is sufficient.

**Source:** CONTEXT.md D-13. Collapse default ensures hypothesis text is readable first.

### Charts Panel
- 2×2 CSS grid: `grid-template-columns: repeat(2, 1fr)`, gap: 32px.
- Each cell: `<img>` with max-width: 100%, height: auto, border-radius: 8px, box-shadow: 0 1px 4px rgba(0,0,0,0.08).
- Below 640px: `grid-template-columns: 1fr` (single column).
- Chart images fetched lazily: `loading="lazy"` attribute on `<img>`.

**Source:** CONTEXT.md D-14. Individual PNGs chosen over zip bundle (Claude's discretion) — simpler to serve via FastAPI FileResponse, no client-side unzip logic needed.

### Download Buttons (secondary style)
- Background: #ffffff. Border: 1.5px solid #e5e7eb. Border-radius: 6px. Padding: 8px 16px.
- Text: text-primary (#111827), Label 14px, weight 600.
- Hover: border-color #6366f1, text-color #6366f1.
- Targets: "Download Report" → `GET /download/scout_report.md`, "Download CSV" → `GET /download/top_repos.csv`, "Download Charts" (4 individual buttons, one per chart PNG) → `GET /download/chart_{1-4}.png`.

**Source:** CONTEXT.md D-15. Individual PNG download buttons chosen (Claude's discretion) — keeps backend simple, avoids zip creation on server.

### Error State
- Displayed inline below query card, above SSE progress strip.
- Style: background #fef2f2 (red-50), border 1px solid #fca5a5, border-radius: 6px, padding: 12px 16px.
- Content: error heading at Label 14px weight 600 (#ef4444) + 1-line description (e.g., "GitHub API error — rate limit reached") at Body 16px text-primary. + "Retry" button (same secondary style as download buttons) that re-submits the last query.
- Run button is re-enabled when error appears.

**Source:** CONTEXT.md D-08.

### Loading Skeleton (Claude's discretion)
- While SSE events arrive but before results are ready: show 3 skeleton placeholder cards (one per persona tab position) using CSS `@keyframes shimmer` — linear-gradient sweep across #f3f4f6 background. Avoids empty space while pipeline runs.
- Skeleton replaces the results section; hidden once first `synthesis_complete` SSE event fires.

## Accessibility Baseline

- All interactive elements keyboard-navigable (native HTML elements only — no div-based click handlers).
- Persona checkboxes: `<label for>` association required.
- Pill tabs: `role="tablist"` / `role="tab"` / `aria-selected` attributes on tab container and individual tabs.
- Collapsible evidence: `aria-expanded` on toggle button, `aria-controls` pointing to list container.
- Confidence dot rating: `aria-label="{N} out of 5 confidence"` on the dot container.
- Color is never the sole differentiator — each persona tab has a text label in addition to color.
