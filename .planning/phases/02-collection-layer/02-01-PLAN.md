```python
class RepoData(BaseModel):
    name: str                              # "owner/repo" format
    url: HttpUrl                           # Repository HTML URL
    stars: int = Field(ge=0)
    star_velocity: float = Field(ge=-1.0, le=1.0)  # stars_last_30d / total_stars
    commits: int = Field(ge=0)             # Last 30 days
    contributors: int = Field(ge=0)
    issues: int = Field(ge=0)              # Open issues
    topics: list[str] = Field(default_factory=list)
    language: Optional[str] = None

class NewsItem(BaseModel):
    title: str
    url: HttpUrl
    source: str                            # "hackernews" or "tavily"
    score: float = Field(ge=0.0, le=1.0)   # Relevance score
    content: str
    published_at: datetime
```

    Implement `src/agents/collection/github_agent.py` with the following structure:

    **Module-level docstring:** Keep existing docstring format per CONVENTIONS.md.

    **Constants:**
    - `GITHUB_API_BASE = "https://api.github.com"`
    - `_MAX_RETRIES = 3` (for 202 retry on stats endpoints)
    - `_RETRY_DELAY = 2.0` (seconds between retries)

    **Helper function `_github_get(url: str, headers: dict) -> dict`:**
    - Async wrapper using `asyncio.to_thread` around `requests.get` with `timeout=30`
    - Calls `resp.raise_for_status()`
    - Returns `resp.json()`

    **Helper function `_github_get_with_retry(url: str, headers: dict, max_retries: int = _MAX_RETRIES) -> dict | None`:**
    - Calls `_github_get`, checks for HTTP 202 status
    - Retries up to `max_retries` times with `asyncio.sleep(_RETRY_DELAY)` between attempts
    - Returns None if still 202 after all retries (fallback to zero values)

    **Tool function `async def search_github_repos(query: str, language: str = "", min_stars: int = 10) -> dict`:**
    - Docstring: Google style, describes args and returns
    - Builds query string: `query + language:{language} + stars:>={min_stars}`
    - Headers: `Authorization: token {os.environ['GITHUB_TOKEN']}`, `Accept: application/vnd.github+json`
    - URL: `{GITHUB_API_BASE}/search/repositories?q={q}&sort=stars&order=desc&per_page=100` (per D-03: 50-100 repos, 1 page)
    - Extracts from each item: `full_name` -> name, `html_url` -> url, `stargazers_count` -> stars, `language`, `topics`, `fork` -> is_fork, `created_at`
    - Returns `{"repos": repo_list}`

    **Tool function `async def fetch_repo_details(owner: str, repo: str) -> dict`:**
    - Docstring: Google style
    - Fetches star velocity: `GET /repos/{owner}/{repo}/stargazers?per_page=100&page=1` with `Accept: application/vnd.github.star+json`
      - Count stars with `starred_at` within last 30 days
      - Get total stars from `GET /repos/{owner}/{repo}` -> `stargazers_count`
      - `star_velocity = recent_stars / max(total_stars, 1)`, clamped to [-1.0, 1.0] (per COLL-02)
    - Fetches commit activity: `GET /repos/{owner}/{repo}/stats/commit_activity` (uses `_github_get_with_retry` for 202 handling)
      - Sum last 4 weeks of `total` field for 30-day commits. Fall back to 0 if None.
    - Fetches contributors: `GET /repos/{owner}/{repo}/stats/contributors` (uses `_github_get_with_retry`)
      - `len(contributors_data)` for contributor count. Fall back to 0 if None.
    - Fetches open issues: `GET /repos/{owner}/{repo}/issues?state=open&per_page=1` — read total from response headers or use `GET /repos/{owner}/{repo}` -> `open_issues_count`
    - Returns `{"star_velocity": float, "commits": int, "contributors": int, "issues": int}`

    **Agent definition at module level:**
    ```python
    github_agent = LlmAgent(
        name="github_agent",
        model="gemini-2.0-flash",
        instruction="""You are a GitHub repository scout. Given a topic query,
        use the search_github_repos tool to find relevant repositories,
        then use fetch_repo_details to get star velocity, commit activity,
        contributor stats, and issue counts for each repo.
        Return the complete list of repos with all details as JSON.""",
        description="Searches GitHub for trending repositories on a given topic.",
        tools=[search_github_repos, fetch_repo_details],
        output_key="github_results",
    )
    ```

    **Imports:** `asyncio`, `os`, `json` from stdlib; `requests` third-party; `LlmAgent` from `google.adk.agents.llm_agent`; `RepoData` from `src.models.schemas` (for type reference in docstrings only — tool functions return dicts, not Pydantic objects, because ADK serializes tool returns).
